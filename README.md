# spring_boot
스프링부트입문

smartsuite-core :동기, 비동기구성
st-resources.js: 라이브러리를 호출방식에 따른 구분
st-js: 프레임워크 클라이언트단에서 사용/정의하는 공통부분들을 정의
sc-preloader: webapp내에서 프레임워크 동작간 필요한 리소스 호출, 모듈 import 수행
sc-mdi: 전체화면구성을 담당하는 요소

테넌시 (테넌트) (ten_id)
하나의 시스템에 대해 A회사도 쓰고 B회사도 똑같이 쓰는 경우
각 디바이스별 데이터를 다르게 보여주기 위한 구분값 

다국어모듈: 최초 서버 구동시 다국어모듈 테이블 내의 모든값을 조회, 이후 캐시형태로 관리 각 값에 대해 설정한 언어별로 매핑

자원설정: WSDL, REST 두가지 방식
첨부파일 구성: 전처리기, 후처리기, 첨부파일 서술관리자
전처리기: 첨부파일이 저장되기전 파일의 추가적인 정보를 설정하는 부분
후처리기: 저장 후 실행되는 로직들(ex DRM 해제)
첨부파일 서술관리자: 첨부파일에 대한 정보 관리

스케쥴러 - Spring Quartz 기반, 실행주기: cron
작업명/그룹, 트리거명/그룹: Quartz 라이브러리 내에서 작업을 구분하는 임의의값
그외에는 그냥 스프링이랑 동일

데이터베이스: datasource-context.xml
